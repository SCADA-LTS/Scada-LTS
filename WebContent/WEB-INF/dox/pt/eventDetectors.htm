Visão Geral

O propósito dos detectores de valores é determinar se o valor de um determinado data point satisfaz uma ou mais condições e se o faz, se tornar ativo e disparar um evento que poderá ser então tratado. Um data point pode ter um número ilimitado de detectores de valores. A maneira de detecção depende do tipo de dado do data point, portanto somente os tipos aplicáveis ao data point em questão constam na lista.

Detectores de valores podem detectar vários tipos diferentes de condições, mas só se aplicam a um único ponto. Se você necessita de condições para vários pontos ou do tipo agendadas pode utilizar detectores de eventos compostos.Veja a documentação de eventos compostos para mais informações.

Um data point pode possuir quanto detectores forem necessários, inclusive vários detectores do mesmo tipo. (i.e. vários do tipo limite superior.)

Para adicionar um novo selecione o tipo na lista e clique em . Note que você precisa clicar no botão "Salvar" para que os detectores sejam realmente associados ao data point. Para excluir um existente, clique em .

Cada detector tem um nível de alarme associado. Isso determina se o detector iniciará um alarme quando ativo e a gravidade do mesmo. Veja a página de ajuda geral para mais informações sobre alarmes.

Limite Superior

Esse detector só se aplica a data points do tipo numérico. O limite máximo é o valor acima do qual o detector se tornará ativo. O detector pode ser opcionalmente configurado com uma duração; o valor do data point deverá permanecer acima do limite por pelo menos o tempo de duração especificado para que o detector se torne ativo. Eventos disparados por este detector automaticamente retornarão ao normal quando a condição não for mais satisfeita.

Limite inferior

Esse detector só se aplica a data points do tipo numérico. O limite mínimo é o valor abaixo do qual o detector se tornará ativo. O detector pode ser opcionalmente configurado com uma duração; o valor do data point deverá permanecer abaixo do limite por pelo menos o tempo de duração especificado para que o detector se torne ativo. Eventos disparados por este detector automaticamente retornarão ao normal quando a condição não for mais satisfeita.

Mudança

Este detector se aplica a todos os tipos de dados. Iniciará um evento quando o valor do data point alterar. Eventos disparados por este detector não possuem retorno ao normal automático.

Sem mudança

Este detector se aplica a todos os tipos de dados. Iniciará um evento caso o valor do data point não se altere por determinado tempo, especificado no campo duração. Pode ser útil, por exemplo, na detecção de equipamentos com falhas. Eventos disparados por este detector automaticamente retornarão ao normal quando a condição não for mais satisfeita.

Sem atualizações

Este detector se aplica a todos os tipos de dados. Iniciará um evento caso o valor do data point não seja atualizado por determinado tempo, especificado no campo duração. (uma mudança no valor de um data point requer que o novo valor seja diferente do anterior. Uma atualização no data point apenas requer que o timestamp seja atualizado, mesmo o seu valor continuando o mesmo). Pode ser útil, por exemplo, na detecção de equipamentos com falhas. Eventos disparados por este detector automaticamente retornarão ao normal quando a condição não for mais satisfeita.

Estado

Esse detector se aplica a tipos binários, múltiplos estados e alfanuméricos. Se torna ativo assim que o data point se encontre no estado especificado. O detector pode ser opcionalmente configurado com uma duração; o valor do data point deverá permanecer no estado determinado por pelo menos o tempo de duração especificado para que o detector se torne ativo. Eventos disparados por este detector automaticamente retornarão ao normal quando a condição não for mais satisfeita.

Contagem de troca de estados

Esse detector se aplica a tipos binários, múltiplos estados e alfanuméricos. Se torna ativo caso o valor do data point mude pelo menos um número de vezes especificado na duração determinada. Eventos disparados por este detector automaticamente retornarão ao normal quando a condição não for mais satisfeita.

CUSUM Positivo

Esse detector só se aplica a tipos numéricos. O limite positivo é a CUSUM (soma acumulada) acima da qual o detector se tornará ativo, e o peso é a constante utilizada no cálculo da soma. O detector pode ser opcionalmente configurado com uma duração; o valor do data point deverá permanecer acima do limite determinado por pelo menos o tempo de duração especificado para que o detector se torne ativo. Eventos disparados por este detector automaticamente retornarão ao normal quando a condição não for mais satisfeita.

CUSUM Negativo

Esse detector só se aplica a tipos numéricos. O limite negativo é a CUSUM (soma acumulada) abaixo da qual o detector se tornará ativo, e o peso é a constante utilizada no cálculo da soma. O detector pode ser opcionalmente configurado com uma duração; o valor do data point deverá permanecer abaixo do limite determinado por pelo menos o tempo de duração especificado para que o detector se torne ativo. Eventos disparados por este detector automaticamente retornarão ao normal quando a condição não for mais satisfeita.
