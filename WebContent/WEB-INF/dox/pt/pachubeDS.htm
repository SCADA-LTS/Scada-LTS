Descrição

Este data source é utilizado para coletar informações genéricas de Pachube. Os dados são organizados no Pachube em feeds, os quais são coleções de fluxos de dados. Uma vez que o fluxo de dados é mapeado para um ponto no ScadaBR, a definição de um ponto Pachube é feito especificando os feeds e o id do fluxo. Este é um polling data source.

Configuração

Todo data source requer um Nome, o qual pode ser qualquer descrição. O Período de atualização determina com que frequência os pedidos HTTPs ao Pachube são realizados. Note que - dependendo da sua conta no Pachube - restrições quando aos pedidos à API podem ser aplicadas. Veja http://community.pachube.com/api#rate_limit para detalhes.

A Chave API é necessária para acessar dados privados. Você pode encontrá-la em na área "my settings" quando você loga na sua conta Pachube. Os campos Timeout e Retry determinam quanto tempo o data source deve esperar pela resposta do servidor web, e quantas vezes ele deve tentar novamente a conexão nos eventos de falhas. Se todas as tentativas forem determine how long to wait for the web server to return the data, and how many times to retry the connection in the event of failures. If all of the retries are esgotads, um evento de exceção do data source é levantado.
