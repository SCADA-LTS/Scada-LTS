Geral

Usuários representam cada indivíduo que usa o sistema. Vários privilégios podem ser associados a usuários para garantir a eles a habilidade de ler e alterar dados e características do sistema.

Há dois níveis básicos de usuários: administradores (), e outros usuários . (). Administradores possuem acesso a todas as funcionalidade do sistema. Outros usuários possuem vários privilégios de acordo com a configuração.

Editando Usuários

Para adicionar um novo usuário, clique no ícone. O Usuário não será adicionado até ser salvado corretamente. Para editar um usuário existente, clique no nome do usuário na lista de usuários.

Senhas não são recuperáveis desde que sejam salvas; elas só podem ser resetadas especificando uma nova senha.

Usuários não podem ser deletados devido a referencias com outros sistemas de objetos que o usuario possa ter. Um usuário desativado não tem a permissão de logar no sistema.

Para testar um endereço de email de usuário, clique no ícone. Isto fará com que um email seja enviado para um endereço de email especificado.

Usuário podem ter notificação de alarme enviados para seu endereço de email sem explicitamente ser específicado um gerenciador de eventos de email. Selecionando Send alarm emails nível de alarme acima "nenhum" fará essas notificações serem enviadas quando o nível de alarme de evento é maior ou igual ao mínimo nível de alarme de evento do usuário, porém somente quando o evento "aumentou" (i.e. no escalation or inactive notifications). Veja também que se o usuário já esta especificado no gerenciador de eventos de email, não serão enviados multiplos emails.

Privilégio de Usuário

O checkbox Administrador determina se o usuário é um administrador.

Se o usuário não é um administrador, todos os "data sources" para o cliente serão listados. Selecionado a caixa perto do nome do "data source" indica que o usuário pode alterar a configuração do "data source", como também todos os dados produzidos por esse "data source".

Se o "data source" não está selecionado, uma lista de todos os "data points" dentro do "data source" serão mostrados. Cada "data point" pode ser individualmente setados para nenhum acesso, acesso de leitura, ou—se o "data point" está setado—acesso de escrita. (Possuir acesso a escrita implica em acesso a leitura.)
