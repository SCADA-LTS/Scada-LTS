Resumo

Modelos de relatórios proporcionam uma definição de como criar instâncias de relatórios. Para uma descrição de modelos vs instâncias, veja a documentação de "Instâncias de relatórios".

Lista de modelos de relatório

Para adicionar um novo modelo de relatório, clique no ícone . Para editar um modelo existente, clique no nome do modelo na lista de "Modelos de relatórios".

Editando detalhes de modelos de relatórios

O nome do modelo é usado como uma referência visual do modelo. Recomenda-se a utilização de um nome único para cada modelo.

Utilize a lista de data points para selecionar os pontos que devem ser incluídos no relatório. Para adicionar data points, selecione o ponto desejado na lista e clique no ícone . Para remover um ponto existente, clique no ícone associado ao ponto.

O valor Date range é utilizado para determinar uma faixa de tempo utilizada para filtrar valores que serão utilizados no relatório. Essa faixa pode ser relativa ou absoluta. Na maioria dos casos uma faixa relativa é apropriada. Faixas de tempo relativas podem ser baseadas tanto em dados "anteriores" como em dados "passados". Uma faixa passada inclui o intervalo de tempo iniciando no valor passado até o instante em que o relatório for gerado. Um faixa anterior também inclui o valor de tempo passado, mas seu tempo final é quantificado para o tempo correspondente. Por exemplo, se o período de tempo for "Hora(s)", o número de períodos for igual a 1 e o relatório for gerado às 18h05. O intervalo de tempo considerado será das 17h00 (inclusive) até às 18h00 (exclusive). Se o número de períodos for, por exemplo, 3, o intervalo será das 15h00 até as 18h00. De maneira similar, o intervalo para "Mes(es)" inicia à meia noite do primeiro dia do mês anterior e encerra no último dia do mês anterior, para o caso do número de períodos ser igual a 1. Outros tipos de períodos funcionam da mesma maneira. Uma semana começa à meia noite de Segunda-feira de acordo com o padrão ISO.

Agendamento de Relatórios

Relatórios podem ser "agendados" para serem gerados automaticamente. Use a seleção Run every... para determinar um simples evento de tempo a partir do qual o relatório será gerado, ou defina uma rotina de acordo com o padrão cron para controle mais específico (veja a documentação de "Cron patterns" para mais informação). Eventos de tempo ocorrem no início do período compreendido. Semanas iniciam Segunda-feira de acordo com o padrão ISO. Um Run delay pode ser aplicado se se os dados esperados para o relatório tendem a serem coletados após o tempo de execução absoluto do relatório.

Relatórios agendados para usuário desabilitados não serão gerados.

Envio de relatórios por email

Apesar de os relatórios não poderem ser explicitamente compartilhados pelo sistema, podem ser implicitamente compartilhados por meio da criação de uma lista de envio de emails para a qual o sistema enviará as instâncias de relatórios gerados. O conteúdo desse email é o mesmo da janela "gráfico de relatório" aberta no painel de Relatórios. Para incluir o arquivo de exportação com o formato CSV no email, selecione a caixa Include tabular data.

Selecione os Email recipients para os quais enviar o email com o relatório. Os destinatários podem ser listas de discussão, usuários do sistema ou endereços de email digitados. Clique no ícone para enviar uma mensagem de teste para os destinatários selecionados.

Importante: Instâncias de relatórios enviadas por email são automaticamente apagadas depois de serem enviadas.

Gerenciamento de modelos

Para salvar um modelo de relatório clique no ícone no canto direito superior do painel de criação de modelos. Para apagar um modelo existente, clique no ícone . Para gerar um relatório imediatamente a partir de um modelo, clique no ícone . Note que alguns modelos de relatórios podem incluir uma grande quantidade de informações e, por causa disso, podem levar algum tempo para serem gerados. Assim, todos os relatórios são gerados de maneira assíncrona da interface de usuário. Quando um ícone é clicado, a instância de relatório será incluída em uma fila onde o usuário pode monitorar seru progresso.
