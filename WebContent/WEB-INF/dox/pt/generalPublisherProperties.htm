Resumo

Editores podem ser considerados os opostos de data sources. Ao invés de coletar dados para o sistema, os editores a distribuem. Isto pode ser útil quando o ScadaBR é utilizado para coletar informações de diferentes sistemas para modificar, filtrar ou gerenciar de alguma maneira. Neste sentido, o ScadaBR não é apenas M2M, mas M2M2M. Editores também podem ser usados para enviar dados a outras instâncias do ScadaBR — por exemplo, para consolidar dados de várias instâncias remotas. Cada instância do ScadaBR pode usar editores para controlar outras de várias maneiras.

Configuração comum de editores

Cada editor tem um Nome que é usado para sua identificação em listas. É recomendável, porém não obrigatório, que cada editor tenha um nome único. A caixa de seleção Habilitado define se o editor está habilitado ou não.

Editores usam reservas de memória para garantir que cada atualização de pontos seja remetida ao destino correto. Se o destino, por qualquer razão, não aceitar essa entrega por um período de tempo considerável, estas reservas de memória podem se tornar muito grandes. Atualizações de pontos não entregues nunca são descartadas da reserva de memória (exceto na desabilitação do editor), mas um alarme — por exemplo, Cache warning size — pode ser definido para ser invocado sempre que o valor de memória seja excedido. Este alarme retorna ao normal quando o tamanho da reserva de memória não mais excede o valor definido, ou quando o editor é desativado.

Um valor de Atualização de evento determina se todos as atualizações de pontos são entregues, ou apenas alterações nos pontos.
