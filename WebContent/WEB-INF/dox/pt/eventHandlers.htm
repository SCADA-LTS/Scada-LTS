Visão Geral

Tratadores de eventos possibilitam ao usuário definir o comportamento do sistema frente ao disparo de um evento. Eventos podem ser disparados devido a várias condições (veja a seção de eventos na página de ajuda geral para mais informações). Quando isso acontece, tratadores de eventos associados são executados.

Há dois tipos de tratadores de eventos:

  • Notificações(e-mail): enviam e-mail para uma lista
  • Set point: escrevem valores em data points (que devem ser configurados como setáveis)

Tipos de Eventos

A seção de tipos de eventos mostra todos os tipos de eventos disponíveis em uma "árvore". A parte de detectores de valores contem uma lista de todos os data points (indicados como ) no sistema que possuem detectores associados. Os detectores de evento são listados abaixo de suas respectivas áreas.

Tipos de eventos para eventos agendados, detectores de eventos compostos, e eventos de sistemas são listados são listados abaixo de sua respectiva parte. A parte de eventos de data source contem uma lista de todos os data sources que podem disparar eventos. Os eventos são listados abaixo de seus respectivos data sources.

Para adicionar um novo tratador de eventos, selecione o tipo que deseja; a seção de detalhes do tipo selecionado será mostrada. O tratador não será salvo até você clicar em . Para editar um tratador já existente, clique no nome do mesmo na lista.

Quando você adiciona um novo, você deve selecionar o tipo do tratador. Uma vez que você o salvou, seu tipo não é mais editável.

Editando (notificações)

Notificações de email mantém duas listas de recebedores. A primeira é a lista para a qual uma mensagem é enviada quando o evento é disparado. A segunda é uma lista opcional para enviar "escalations".

Para adicionar uma lista de email para uma lista de recebedores, selecione-a da lista de adicionar recebedores e clique em . Para adicionar um usuário, selecione um usuário da lista Add user e clique em . Para adicionar um outro endereço qualquer, entre com o mesmo no campo adicionar endereço e clique em . Para excluir qualquer recebedor da lista, clique em . Para testar uma lista de recebedores, clique em .

Para ter email de "escalation" enviados, selecione o campo Send escalation, e entre com Escalation recipients. O "escalation" email será somente enviado se o evento associado estiver ativo para o período Escalate if active for

Um evento inativo de notificação pode ser enviado selecionado o Send inactive notification campo. Se selecionado, a notificação será enviada para todos os recebedores de eventos uma vez que o evento se tornou inativo. Note que, se uma email de "escalation" não foi enviado, a notificação inativa não será enviada para a lista de "escalation". Somente aqueles recipientes que receberam uma notificação ativa irão receber uma notificação inativa.

Editando (set point)

Quando um evento é disparado, esse tratador irá escrever o valor em um dado data point. O target é o data point que terá seu valor alterado. Opcionalmente, o data point pode ser setado com o valor que alterou o evento, porem isso é possível somente se o evento foi disparado por um detector de valores do tipo "value changed", e o tipo de dado do caminho é o mesmo que o tipo de dado da fonte. Para setar o caminho com o valor da fonte, selecione o campo Use source value. Ou então, entre com o valor que deseja escrever no campo Value to set.
