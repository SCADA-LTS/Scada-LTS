Comandos

Para pontos do tipo OUTPUT, devem ser definidos quais os parâmetros do comando que serão utilizados como padrão caso não sejam especificados na hora da execução do comando. Para pontos do tipo BYNARY OUTPUT, os comandos válidos (que devem ser executados na watchlist no ícone apropriado, como nos outros datasources) são os seguintes:

pon/poff -> pulse on/off

lon/loff -> latch on/off

close/trip -> close/trip

Todos os comandos podem ser seguidos por mais dois parâmetros, que serão o TIME ON e o TIME OFF do comando em milissegundos. Exemplo: pon,500,200

Pode-se também entrar apenas com "1", "0", "true" ou "false". Neste caso, o comando enviado será o comando padrão (especificado para o ponto quando cadastrado).

Exemplo

Ponto binário cadastrado como CLOSE/TRIP, TIME ON 1000 e TIME OFF 0

1 -> o sistema enviará o comando CLOSE com TIME ON 1000 e TIME OFF 0

0 -> o sistema enviará o comando TRIP com TIME ON 1000 e TIME OFF 0

pon -> o sistema enviará o comando PULSE ON com TIME ON 1000 e TIME OFF 0

poff,300,0 -> o sistema enviará o comando PULSE OFF com TIME ON 300 e TIME OFF 0
