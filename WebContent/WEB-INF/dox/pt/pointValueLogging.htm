O ScadaLTS suporta quatro tipos de registro de valores do ponto.

Quando o valor do data point muda é a configuração padrão de registro. O valor do ponto e seu tempo de ocorrência é escrito na base de dados somente se o valor do ponto realmente mudou. Esta configuração fornece o melhor compromisso de qualidade de informação histórica vs eficiência de espaço de armazenamento. Para pontos Numéricos, uma Tolerância ou "banda morta", pode também ser definida; o valor será registrado se o valor absoluto da diferença entre o valor atual e o valor anterior for igual ou maior que a tolerância definida.

A opção Todos os dados faz com que toda atualização do ponto seja salva na base de dados. Somente use esta opção se o conhecimento de toda atualização é definitivamente requerida, já que uma grande quantidade de dados pode ser acumulado muito rapidamente, o qual com o tempo, pode causar serias degradações na performance do sistema.

A opção Não registrar previne qualquer dado histórico do ponto de ser armazenada na base de dados. Por favor note que o renderizador de gráfico tipicamente requer dados registrados para fucnionar apropriadamente. Somente use a opção "Não registrar" quando tiver certeza de que o valor histórico não é necessário.

A opção Intervalo permite que a coleção de dados via data source seja separada de seu registro. Use esta opção quando o data source estiver coletando dados muito frequentemenet (por exemplo, para propósitos de detecção de eventos), mas todos os dados não precisam ser registrados. O Tipo de valor pode ser escolhido com essa opção. "Instantâneo" significa que o valor atual do ponto é registrado, enquando as outras opções calculam estatisticas dos valores do ponto sobre o período de registro.

A opção Quando o valor do timestamp muda tem comportamento similar ao "Quando o valor do data point muda, mas o timestamp da amostra é comparado ao invés do valor.

O descarte de valores podem ser definido para lidar com equipamentos que ocasionamente enviam valores errados. Quando Descartar valores extremos é selecionado, os campos Descartar limite inferior e Descartar limite superior são abilitados. Valores menores que o limite inferior ou maiores que o limite superior surgidos posteriormente serão descartados. Qualquer valor existente já armazenado será mantido.

A opção Descartar determina quanto tempo os valores históricos do ponto devem ser mantidos na base de dados. Os valores podem ser mantidos praticamente quanto tempo for necessário, mas note que uma degradação na performance do sistema pode ocorrer se o histórico de um dado ponto se tornar muito grande. (Na verdade, a performance do sistema depende fortemente do hardware do host.) O processo de descarte é executado automaticamente toda manhã às 3:05am do tempo local.

O Tamanho de armazenamento padrão é o número de valores que serão armazenados na memória. Esta configuração é especialmente importante quando um método que não "Todos os dados" é usado. Valores que são armazenados na memória são confrontados os armazenados na base de dados quando necessário. Definir um valor muito alto para esta configuração pode melhorar a performance do sistema (visto que ela pode prevenir um acesso à base de dados), mas deve-se ter cuidado pois a memória é consumida pela cache, e problemas relacionados à falta de memória podem ocorrer.
