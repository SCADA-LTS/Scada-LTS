Descrição

O data source Galil é usado para trocar dados com equipamentos Galil DMC-21x2 acessados sobre uma rede I/P. Os equipamentos podem estar disponíveis em uma rede local ou intranet, bem como podem estar em qualquer lugar na internet. Este é um data source do tipo polling.

Configuração

Todo data source deve ter um Nome, que pode ser qualquer descrição. As configurações de Host e Porta determinam como localizar o equipamento na rede. O host pode ser um nome de domínio ou um endereço IP. Os campos Timeout e Retries determinam o comportamento do sistema no caso de uma falha no polling. O data source aguarda por um timeout com o número fornecido de milissegundos por uma resposta do equipamento. Se não houver resposta, a solicitação será o número de vezes determinado. O Período de atualização determin com que freqüência o equipamento é solicitado por dados.

Uma vez que a conexão é estabelecida com o equipamento, ela é mantida aberta por tanto tempo quanto o equipamento permitir. Conexões fechadas forçam uma tentativa automática de abertura de uma nova conexão.

Testes de comandos

A conectividade e estado de equipamentos podem ser testados usando a seção de comando de testes Galil. Esta funcionalidade abre uma nova conexão para enviar o comando, então não irá conflitar com qualquer data source em funcionamento. Entretanto, se o equipamente apenas permitir um número limitado de conexões, a conectividade pode falhar.

O valor Comando pode ser qualquer string de comando (excluindo o finalizador CRLF e opcionalmente excluindo um ponto-e-vírgula de fechamento). O resultado do comando será mostrado abaixo do botão Executar. Se a conexão com o equipamento falhar, uma mensagem de erro será mostrada. A conexão irá expirar após a duração do timeout se nenhuma resposta for recebida. Se uma resposta de erro for recebida (como, por exemplo, se '?' for recebida), será mostrada a mensagem "(error response)".
