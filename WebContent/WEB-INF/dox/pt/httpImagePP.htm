A URL é o endereço da imagem a ser recuperada. Use o ícone para testar a URL, que será aberta em uma nova janela ou aba do navegador. Os campos Timeout e Retry determinam o tempo de espera para o servidor de web retornar a imagem, e quantas vezes deve tentar novamente a conexão no caso de falhas. Se todas as tentativas forem gastas, uma exceção do tipo data point é levantada.

Imagens podem ser automaticamente diminuídas conforme solicitadas para economizar memória ou espaço no sistema de arquivos. Imagens também podem ser aumentadas, mas como isso não melhora a resolução, normalmente não é prático fazer isso. O "percentual" do tipo de dimensionamento é um método de dimensionamento "relativo" que resulta em uma imagem dimensionada ao tamanho Percentual da escala da imagem original. O método de dimensionament "caixa" é um método de dimensionamento "absoluto" que redimensiona a imagem original de maneira a caber na "caixa" definida pela Largura de escala e altura de escala.

O limite de leitura permite a definição de um tamanho máximo de respota que pode ser recebido. Se o tamanho da imagem exceder o valor definido, os dados serão descartados e um evento de recuperação de dados é levantado.

Se a fonte da imagem for uma web cam é possível passar o código de transmissão ao vivo na área de texto proporcionada. Este código será entrado literalmente em uma nova janela ou aba do navegador (cercado por tags html e body). Se a transmissão ao vivo da web cam necessita de componente ActiveX ou um applet Java, estes deverão ser instalados apropriadamente dentro da instância do ScadaBR.
