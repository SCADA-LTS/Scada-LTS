Visão Geral

Data points em um data source virtual geram dados sem nenhuma influência externa, sejam aleatórios ou determinísticos. Qualquer tipo de dado suportado pode ser gerado e várias opções para o modo como os valores serão alterados estão disponíveis.

Atributos comuns a todos os data points virtuais são:

  • Nome: o nome do ponto para identificação.
  • "Setável": permissão se o ponto pode ou não ser alterado manualmente.
  • Tipo do dado: o tipo de dado. Veja mais abaixo para detalhes.
  • Tipo da troca: o algoritmo que determina como os valores serão alterados. Veja mais abaixo para detalhes.
  • Valor inicial: o valor inicial para quando o data source for habilitado.

Data points binários

Valores binários possuem apenas dois estados possíveis: ligado ou desligado, verdadeiro ou falso, etc. Existem três tipos de alteração desses valores possíveis:

  • Alternada: o valor alterna entre seus estados a cada ciclo.
  • Sem alterações: o valor não altera automaticamente (útil para data points setáveis manualmente)
  • Randômica: o valor muda randomicamente (somente a cada ciclo)

Data points de múltiplos estados

Pontos de múltiplos estados permanecem em apenas um de seus múltiplos estados possíveis: como ligado, desligado e habilitado. Os valores possíveis de estado para cada ponto devem ser definidos no campo de valores. Existem três tipos possíveis de alteração de seus valores:

  • Incremental: o ponto passa por todos os estados possíveis. Se Roll está habilitado, o data point retornará ao valor inicial uma vez que o mesmo alcançar seu último valor. Se Roll está desabilitado, the point will pass through all of the points in reverse before starting over at the beginning.
  • Sem alterações: o valor não altera automaticamente (útil para data points setáveis manualmente)
  • Aleatória: o valor muda randomicamente (somente a cada ciclo)

Valores numéricos

Valores numéricos são valores analógicos, e podem ser qualquer número real capaz de ser representado por um ponto flutuante de 64 bits. Existem cinco tipos possíveis de alteração de seu valor:

  • Brownian: alteração browniana aleatória — o ponto irá mudar pe acordo com uma variação diferencial aleatória de seu valor anterior. Limites mínimos e máximos para o valor podem ser definidos em Mínimo e Máximo respectivamente. A variação máxima determina o maior diferencial aleatório que pode ser gerado.
  • Incremental: o ponto altera do número alteração especificado a cada ciclo. Este valor pode ser negativo. Os valores máximos e mínimos podem ser determinados. Se Roll está habilitada, o valor retornará ao limite uma vez que exceda o limite oposto. Se Roll está desabilitada, o data point utilizará o valor de alteração inverso uma vez que exceda os limites.
  • Sem alterações: the point does not automatically change (useful for points that are manually settable)
  • Aleatório: o ponto assume uma valores contidos em uma distribuição uniforme entre os valores máximo e mínimo, incluindo esses limites.
  • Atração: o ponto se aproxima de maneira assintótica a um outro ponto no sistema. A variação máxima determina quão grande pode ser o incremento em direção ao ponto—alvo. A volatilidade pode ser usada para representar flutuações aleatórias no ponto.

Pontos alfanuméricos

Pontos alfanuméricos representam seqüências de caracteres e, assim, não estão sujeitos a alterações significativas por algoritmos. Assim, o único tipo de alteração possível é sem alteração.
