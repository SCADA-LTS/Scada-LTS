Descrição

O objetivo de um renderizador de texto é converter um valor bruto em um formato que é mais reconhecível e informativo a um usuário humano. As opções de renderização dependem do tipo de dado do valor a ser renderizado, de tal maneira que somente os tipos de renderização que se aplicam ao tipo de dado do ponto em questão são listadas na seleção Tipo.

Binário

O processador Binário permite ao usuário definir um rótulo para representar cada um dos dois estados binários (0 e 1). Qualquer rótulo pode ser definido. Além disso, pode-se definir uma cor distintiva para cada estado. Exemplos típicos de rótulos são off/on, desligado/ligado, parado/rodando ou falso/verdadeiro, mas qualquer texto pode ser utilizado.

O processador Simples é o renderizador padrão de binários. Ele mostra os valores brutos inalterados, ou seja, 0 e 1.

Numérico

O renderizador analógico permite ao usuário definir um Formato usado para determinar coisas como um numero de casas decimais mostradas, e The Analog renderer allows the user to define a Format used to determine things like the number of decimal places to display, and se as casas de milhares devem ser delimitados. (Veja a documentação "Formato Numérico"). Um Sufixo pode também ser adicionado para mostrar as unidades do valor. Sufixos podem conter entidades HTML como ° (escrito como '&deg;') para visualização em páginas HTML. Veja http://www.w3.org/TR/REC-html40/sgml/entities.html para mais informações sobre entidades HTML.

O processador Simples é o renderizador padrão de numéricos. Mostra o valor bruto inalterado.

O processador Faixa permite ao usuário definir um Formato numérico (veja a documentação "Formato Numérico" para maiores informações), bem como um numero arbitrário de valores textuais para representar diversas faixas numéricas "de/até". A cada uma destas faixas pode ainda ser atribuída uma cor distinta. O formato numérico só é utilizado se o valor não se encaixa em nenhuma faixa definida. Por exemplo, a faixa "frio", "normal" e "quente" podem ser definidas para um ponto de temperatura, com cores apropriadas para cada uma.

O processador Tempo permite ao usuário definir um Formato (veja a documentação "formato de data/hora" para mais informação). Um formato típico de data seria "MMM d, yyy". UM típico formato de hora seria "HH:mm:ss". Formatos de Data e hora podem ser combinados como for necessário. Um expoente de conversão de milissegundos pode ser também definido. Este é um expoente em potência de 10 que será usado para converter um dado valor ao equivalente em milisssegundos. Por exemplo, para converter de segundos use 3, de microssegundos, use -3. Se o valor já está em milissegundos, use 0. Os valores de tempo são assumidos para iniciar em 0. em 1ยบ de Janeiro, 1970 à meia noite.

Multi-estados

O processador Simples é o processador padrão de multi-estados. Mostra os valores brutos inalterados.

O processador Multi-estados permite ao usuário definir um numero arbitrário de valores textuais para representar diferentes pontos de estados. A cada um desses estados pode-se também assumir uma cor distinta. Se um estado para o qual não foi definido um rótulo ocorrer, o valor bruto é mostrado. Rótulos típicos são ligado/desligado/desabilitado, ou desligado/quente/frio.

Alfanumérico

O processador Simples é o único processador alfanumérico. Mostra o valor bruto inalterado, incluindo um Sufixo opcional.
