O Tipo do dado é a representação do valor no ScadaBR. Dados recebidos são automaticamente convertidos ao tipo de dado requerido. O Valor RegEx determina onde no conteúdo do email encontrar o valor do ponto. Veja a documentação "Expressões Regulares" para mais informações. Note que somente a primeira instância da expressão encontrada no conteúdo é usada.

Para o tipo de dado binário, o Valor 0 binário pode ser definido. Se o valor encontrado no conteúdo casar com ele, o ponto será atualizado com 0. Caso contrário, o ponto é atualizado com 1. Para o tipo de dado numérico um Formato Numérico pode ser definido, o qual fornece informação de como o texto encontrado pela RegEx deve ser analisado. Veja a documentação "Formatos Numéricos" para mais informações.

Uma RegEx de tempo pode ser definida para localizar o timestamp do valor no conteúdo retornado. Se a RegEx for definida, um Formato de tempo também deve ser definido, o qual fornece informação de como o texto encontrado pela RegEx deve ser analisado. Veja a documentação "Formado de data/hora" para mais informações.

Voce pode testar sua RegEx entrando com uma amostra de conteúdo de um email na área Testar conteúdo e clicando no ícone associado .

Exemplo

Digamos que a mensagem alvo se pareça com isto: (envelope removido):

GPGGA,123519,4807.038,N,01131.000,E,1,08,0.9,545.4,M,46.9,M,,

... e você quer encontrar o nono campo (ou "545.4", contando a partir de "GPGGA"), você poderia utilizar esta regex:

GPGGA,.*?,.*?,.*?,.*?,.*?,.*?,.*?,.*?,(.*?),.*

Esta abordagem permite a seleção arbritária de mensagens de acordo com outro conteúdo. Por exemplo, se a parte "123519" indicasse um id de uma estação metereológica em particular (assumindo que você possuisse várias), sua regex para selecionar apenas mensagens desta estação poderia ser:

GPGGA,123519,.*?,.*?,.*?,.*?,.*?,.*?,.*?,(.*?),.*

All other GPGGA messages from other stations would be ignored.

GPGGA,123519,.*?,.*?,.*?,.*?,.*?,.*?,.*?,(.*?),.*

Todas as outras mensagens GPGGA de outras estações seriam ignoradas.
