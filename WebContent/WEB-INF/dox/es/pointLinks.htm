Descripción General

Los enlaces a puntos son utilizados para actualizar los puntos de destino basado en el valor de los puntos de origen. Los dos puntos dos puntos se mantienen "sincronizados", permitiendo utilizar una fórmula para transformarlo si se requiere. Esencialmente, el punto origen puede ser utilizado para leer los valores en el sistema (de una o más fuentes de datos) e inmediatamente actualizar los valores en otro sistema (a una o más fuentes de datos).

El enlace a punto también se puede obtener con el uso de evento por cambio o actualización en los puntos de origen y, posteriormente, crear un punto en el manejador de eventos para fijar en cada punto destino el nuevo valor. Sin embargo, utilizar el enlace a punto tiene algunas ventajas para este propósito:

  • Los enlaces a puntos no levantan eventos por lo que no sobrecargan el sistema, con lo que mejorará el rendimiento
  • Los enlaces a puntos permiten la definición y utilización de un script opcional que puede transformar el valor original como se requiera
  • La interface de los enlaces a punto es mostrada en una página sencilla, con lo su configuración es mucho más fácil

Nota: los enlaces a puntos pueden haber sido configurados de forma de lazos infinitos. Esta condición será detectada por el sistema en tiempo de ejecución, y este detendrá su propagación y levantará un evento de falla recursiva.

Lista de enlaces a puntos

Para agregar un nuevo enlace a punto, haga clic en el icono . Para editar un enlace existente, haga clic en la descripción de éste en la lista "Enlaces de Puntos".

Editando los detalles del enlace a punto

El ID de Exportación es el valor utilizado para identificar en enlace del punto a través de múltiples instancias del ScadaBR. Este es utilizado por la función de Importar/Exportar. Este valor debe ser único entre todos los enlaces a puntos. Un valor aleatorio y único será automáticamente asignado por el sistema, pero puede ser cambiado para ser más adecuado a su aplicación.

El Punto de origen es el punto que le será monitoreado su valor. este es el punto desde donde proviene el valor.

El Punto destino es el punto que será configurado con un valor. Este es el punto para donde el valor será enviado.

El Script es un valor opcional que puede transformar el valor del punto origen como se requiera. Para más información de del uso de los scripts puede ser encontrada en la documentación de las propiedades de los puntos Meta. A la variable de contexto se le denomina "source" (origen o fuente). Así, un script pudiera ser "return source.value / 2;" lo que configurará en el punto destino la mitad del valor del punto origen. Note que puede realizar transformaciones en los tipos de datos, por ejemplo "if (source.value > 0) return true; return false;", lo cual puede convertir un valor de tipo numérico en uno de tipo binario. Utilice el icono para validar su script (El punto origen debe estar disponible).

El campo Evento determina si el punto destino es configurado cuando el punto se actualiza (p.e. recibe una nueva estampa de tiempo pero no necesariamente un valor nuevo), o cuando su valor cambia (p.e. recibe una estampa de tiempo nueva y un nuevo valor).

Puede deshabilitar un enlace a punto de forma que este no se realice. Esto puede ser útil cuando prefiera conservar el enlace en lugar de eliminarlo y que este sólo requiere ser habilitado para utilizarlo.

Para almacenar un enlace a punto, haga clic en el icono en la parte superior derecha. Para borrar un enlace existente, haga clic en el icono .
