Descripción General

Esta utilidad le permite realizar exportaciones e importaciones de varios aspectos de la configuración del Scada, con los siguientes beneficios:

Respaldo y Recuperación
Exporte completamente la configuración, y guarde los resultados en formato plano de forma segura. Si la instancia de la aplicación sufre alguna falla fatal, puede crear una nueva instancia e importar la información respaldada desde el archivo de configuración. Así podrá regenerar rápidamente el sistema antes del daño.
Crear y mantener instancias idénticas
Exporte los elementos de configuración a repetir en sus instancias para crear instancias similares. Importe las configuraciones comunes y ajuste según requiera. Para hacer cambios en todas sus instancias, simplemente realice el cambio de configuración, genere el archivo de respaldo e impórtelo en las demás instancias.
Mantenimiento de configuraciones complejas y largas
Aunque es una herramienta simple y de fácil uso, la interface de usuario de Mango no está diseñada para manejar configuraciones extremadamente largas y complejas. En esos casos los archivos de exportación pueden ser utilizados como una forma alternativa para administrar su configuración. Para estos casos realice los cambios directamente en el archivo de configuración, y luego importe sus instancias.

Exportando

Puede exportar la configuración completa, al acctivar las casillas de cada una las opciones.

Seleccione lo desea exportar: Puede elegir las porciones de la configuració que requiere. El resultado de la operación de exportación se mostrará en el área de texto denominada Dato. Desde esta área podrá copiar y pegar código desde, o hasta otro archivo, o si prefiere, realizar los cambios directamente.

Importando

De la misma forma que puede realizar una exportación, puede realizar una importación selectiva (sólo una porción) de toda la configuración del sistema. Las porciones que sean excluidas en la exportación permanecerán sin cambios (inalteradas) al momento de realizar la importación. Por ejemplo, si los datos de importación no incluyen el atributo de "Detector de eventos" de un punto, la lista original de detectores de eventos del punto permanecerá sin cambios. Igualmente, si se incluye el atributo, este será reemplazado.

En general, cualquier objeto del sistema tendrá un identificador único que podrá ser creado y posteriormente actualizado— pero no eliminado. Estos objetos son soportados por esta utilidad y tendrán un Id de Exportación, o XID. Este atributo está específicamente diseñado para proporcionar un identificador único entre los objetos de instancias del sistema. Esta característica es similar a la clave primaria que genera una base de datos, aunque estas claves primarias no pueden hacer pareja con múltiples instancias de Mango. Cuando un objeto —tal como una fuente de datos—es creado utilizando la interfaz de Mango, ésta automáticamente le asignará una instancia única de exportación XID. En el caso que exista más de una instancia de Mango, no hay garantías de que este identificador sea único entre las instancias del sistemas, por lo que hay una alta probabilidad que se repita el valor. Es posible cambiar el XID con cualquier valor de 50 caracteres, y será tan largo como guste, mientras sea único en toda la instancia.

Las importaciones pueden fallar por múltiples razones:

  • Cuando el formato del código JSON no está bién, toda la importación fallará y se mostrará un mensaje de error que describe la causa y la ubicación del problema dentro del texto del código.
  • Cuando un elemento o sus datos no están en el código o son inválidos, El objeto asociado con la importación fallará, pero los demás objetos pueden ser importados de exitosamente. Finalmente obtendrá una lista de los objetos que fueron importados sin problemas, y otra con los mensajes de error que describen las fallas.

El Formato

La posibilidad importar/exportar utiliza el formato JSON (para más información vea http://www.json.org/) como formato de los archivos de configuración. JSON utiliza un formato de datos muy parecido que es muy fácil de entender y utilizar. (Para quienes estén familiarizados con Javascript, JSON les será muy familiar inmediatamente.) JSON fue elegido sobre los más comunes formatos de XML ya que proporciona una mejor conformación de la actual estructura de objetos del sistema, además de ser más compacta.

Mango creará automáticamente archivos de exportación JSON "elegantes", p.e. utilizará formatos de salida con identación y fines de lineas. La cantidad de espacios utilizados para la identación serán controlados por el atributo Nivel de identación.
