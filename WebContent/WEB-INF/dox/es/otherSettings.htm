El rendimiento de la UI corresponde respuesta de la interfaz de usuario. Esto incluye todas las actividades que se administran por el mecanismo de encuestas largas, como el indicador de alarma de nivel máximo, la actualización de la lista de supervisión, actualizaciones de la lista de eventos, etc. Si el servidor de la aplicación tiene poca capacidad de procesamiento (CPU pequeño), puede ajustar el rendimiento de la interfaz de usuario en un nivel menor para disminuir el consumo producido por los navegadores clientes en el CPU del servidor.

Cuando la opción "Enviar errores a Serotonin" está activada, las excepciones observadas por esta instancia de Mango serán enviadas a Serotonin Software para su revisión. Por favor mantenga esta casilla inactiva, si está realizando cambios en el código de la aplicación para sus propios propósitos.

Utilice la configuración de "Purgar eventos" para configurar el tiempo máximo de permanencia de un evento dentro de la base de datos antes de ser eliminado del histórico. Purgar elimina los eventos de forma permanente de la base de datos, al igual que cualquier comentario que haya sido ingresado por los usuarios sobre estos.

Utilice la configuración "Purgar reporte de instancias" para configurar el tiempo máximo que permanecerá un reporte antes de ser eliminado de la base de datos. Al purgar se elimina permanetemente el reporte de la instancia. La mejor práctica es mantener este valor lo más bajo posible ya que los reportes consumen grandes cantidades de espacio de almacenamiento en disco. La acción de purgar los reportes puede evitarse configurando el atributo "Prevenir la purga" en la instancia del reporte.

Haga clic en el botón Purgar todos los Datos para eliminar todos los puntos de ejemplo que estén en la base de datos. Esto es útil cuando se quiere dar a la instalación del sistema un "inicio limpio", especialmente si existen datos "no esenciales" en las actividades de configuración y de instalación.

Utilice la configuración "Descarte valores de puntos fechados en el futuro" para definir la cantidad máxima de tiempo en el futuro que se aceptarán datos nuevos para los valores de los puntos. Entonces al presentarse datos con estampa de tiempo mayor a este valor se descartarán, y se escribirá un mensaje de error en el registro.
