Configuración

Cada punto de dato configurado en la fuente de datos Serie Personalizado se corresponde con un valor recibido desde el dispositivo serie, que se encuentra conectado a un puerto serie del ordenador que aloja este SCADA.

Cada nuevo punto aÃ±adido a esta fuente de datos es identificado mediante un Nombre y un identificador interno del SCADA generado de forma automática, ID de Exportación. Existen dos posibles formas de comunicacón dependiendo de como un dispositivo serie proporciona los datos:

  1. Envío de comandos y recepcón de respuesta
  2. Recepción periódica de datos

Envío de comandos y recepción de respuesta

Si el dispositivo serie precisa ser consultado para proporcionar un dato, es necesario establecer el Número de comandos a enviar al dispositivo serie antes de obtener la respuesta solicitada al dispositivo serie. En el caso de que sólo fuera necesario el envío de un Primer Comando, la respuesta solicitada es proporcionada directamente por el dispositivo serie. En caso de ser necesarios dos o tres comandos para obtener la respuesta deseada desde el dispositivo serie, la respuesta al primer comando es comparada con la Respuesta esperada para primer comando. Si ambas coinciden, un Segundo Comando es enviado y, si un Tercer Comando es necesario, el mismo proceso de comparaciÃ³n es realizado entre la respuesta al segundo comando y la Respuesta esperada para segundo comando. En el caso de que las respuestas del dispositivo serie a los diferentes comandos intermedios coincidan con las esperadas, se almacena en le punto de datos la parte de la respuesta final que coincida con la expresiÃ³n regular configurada (Valor de ExpresiÃ³n Regular (RegEx)), en forma de un valor binario, numérico, multiestado o alfanumérico (Tipo de Dato)

Esta es la forma habitual de comunicación con un dispositivo serie. En cada período de actualización de la fuente de datos se envía al dispositivo serie el Comando configurado en cada punto de datos según el Formato de trama para comando definido en la fuente de datos Serie Personalizado, y se obtiene la respuesta proporcionada por el dispositivo serie a dicho comando.

Recepción periódica de datos

Si el dispositivo serie envía datos de forma periódica sin recibir ningún comando desde la fuente de datos Serie Personalizado, el campo Número de comandos de cada punto de datos debería dejarse con valor cero, indicando de esta forma que la fuente de datos se encuentra configurada para únicamente recibir datos desde el dispositivo (NINGUN COMANDO ES ENVIADO AL DISPOSITIVO SERIE). El valor final a almacenar como punto de dato se selecciona a partir de un Valor de Expresión Regular. Dependiendo del Tipo de dato, este valor se guarda como un dato binario, numérico, multiestado o alfanumérico.
