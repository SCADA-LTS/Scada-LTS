Scada-LTS soporta cuatro tipos de registro de valor puntual

Cuando el valor cambia que es el ajuste por defecto. El valor y su ocurrencia temporal se almacenan en la base de datos sólo si el valor del punto cambia. Este escenario ofrece la mejor relación calidad/espacio de almacenamiento (histórico). Para los valores numéricos, también se puede establecer una Tolerancia, o "zona muerta"; el valor se almacenará si la diferencia entre el valor actual y el anterior es mayor o igual a la tolerancia establecida.

El parámetro Todos los datos hace que todas las actualizaciones de datos se almacenen en la base de datos. Utilice este ajuste sólo cuando sus limitaciones lo requieran, ya que causará rápidamente una gran acumulación de datos y ralentizará seriamente el rendimiento.

El parámetro No registrar impide que este punto sea registrado en la base de datos. Ten en cuenta que en este caso no podrás aplicar ningún tipo de renderizado de gráficos para este punto. Usa "No grabar" cuando estés seguro de que no necesitas el historial de datos.

El parámetro Intervalo el número de accesos al valor del punto son diferentes del número de registros en la base de datos. Utilice este parámetro cuando sea necesario acceder con frecuencia a los datos (por ejemplo, para la detección de eventos), pero no es necesario registrar todos los datos. Tipo de valor permite elegir el valor que se registrará. "Instantáneo" el valor actual del punto, las otras opciones son estadísticas sobre el valor del punto durante el período definido.

El parámetro para cambiar el timestamp. Comparable al parámetro cuando el valor cambia, pero es la marca de tiempo la que se compara y no el valor.

Rechazar valores extremos se utiliza para dispositivos que pueden emitir ocasionalmente valores erróneos. Cuando se selecciona Rechazar valores extremos, se activan los cuadros de diálogo rechazar límite bajo y rechazar límite alto. Todos los valores ya guardados se conservarán.

El parámetro Purga determina la duración de la historia del valor del punto en la base de datos. En la práctica, los valores pueden mantenerse tanto tiempo como sea necesario, pero hay que tener en cuenta que esto puede dar lugar a una degradación del rendimiento del sistema si el historial de un punto se hace demasiado grande (de hecho, el rendimiento del sistema depende en gran medida del equipo informático central). El proceso de purga comienza todos los días a las 3:05 a.m. hora local.

El tamaño de la caché por defecto es el número de valores que se almacenarán en la memoria. Este parámetro es importante cuando se elige un método distinto a "Todos los datos". Los valores en la memoria serán eventualmente comparados con los valores almacenados en la base de datos. Un valor elevado del tamaño de la memoria caché puede mejorar el rendimiento del sistema (ya que puede impedir las consultas a la base de datos), pero la memoria es entonces consumida por la caché y pueden surgir problemas relacionados con la falta de memoria. "Vaciar la memoria caché" no es necesario en teoría, pero puede ser útil en caso de eliminación manual de puntos (utilizando funciones SQL, por ejemplo).
