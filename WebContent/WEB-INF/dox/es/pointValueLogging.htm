El ScadaBR soporta cuatro tipos de registro de valores de punto

La configuración predeterminada del registro es Cuando cambie su valor. El valor del punto y su hora de ocurrencia serán escritos solamente cuando el valor del punto ha cambiado. Esta configuración proporciona la mejor relación de calidad de la información histórica frente a la eficiencia en el uso del espacio de almacenamiento. Para puntos numéricos, su Tolerancia, o "banda muerta", pueden ser definida; el valor será registrado cuando el valor absoluto de la diferencia entre el valor del punto actual y su valor previo es igual o excede la tolerancia definida.

La configuración Toda la data causa que cada actualización del punto será almacenada en la base de datos. Unicamente utilice esta configuración cuando requiera registrar cada actualización del punto, ya que esta configuración genera grandes cantidades de datos que serán almacenados de forma constante y rápida, y con el tiempo puede causar serios problemas de degradación del rendimiento del sistema.

La configuración No registrar omite la escritura de cualquier valor histórico del punto en la base de datos. Por favor note que la generación de gráficos requieren de la data guardada en la base de datos para funcionar correctamente. Sólo utilice opción "No registrar" cuando tenga certeza de que los valores históricos no serán requeridos.

La configuración Intervalo permite el recolectar la data de la fuente de datos de forma separada a como se registra. Utilice esta opción cuando la fuente de datos recolecta la información de sus puntos de forma muy frecuente (por ejemplo, para ser utilizado por detectores de eventos), pero no requiere que toda la data que sea generada sea almacenada. En esta opción también puede elegir el Tipo del valor. "Instantáneo" que el actual valor de punto será registrado, a diferencia de las otras opciones que utilizan métodos estadísticos sobre los valores del punto en el período de tiempo del registro.

La configuración Cuando cambie su estampa de tiempo tiene un comportamiento similar al del cambio del valor del punto pero se compara la estampa de tiempo en lugar del valor.

Puede configurar valores a descartar para interactuar con equipos que ocasionalmente puedan enviar valores errados. Al activar Descartar valores extremos, las casillas del Límite inferior de descarte y Límite superior de descarte serán activadas. Los valores por debajo del limite inferior o mayores al límite superior serán descartados. Nota: Los valores almacenados previamente a esta definición permanecerán en la base de datos.

La configuración Purgar determina cuanto tiempo permanecerá almacenado un valor histórico en la base de datos. Los valores pueden permanecer tanto tiempo como sea necesario, pero note que puede ocurrir una degradación del rendimiento del sistema si el histórico de un punto se torna muy extenso. (El rendimiento del sistema depende de manera directa con las características del hardware del servidor.) El proceso de purga es realizado automáticamente cada mañana a las 3:05am hora local.

El Tamaño de caché predefinido es el número de valores a ser registrados en memoria. Esta configuración es especialmente importante cuando es utilizado un método de registro diferente a "Toda la data". Los valores almacenados en memoria serán cotejados con los almacenados en la base de datos cuando sea necesario. Configurar este valor muy alto puede afectar en el rendimiento del sistema (puede evitar una consulta a la base de datos), pero se debe tener precaución con la memoria es consumida por el caché y con posibles fallos que en esta ocurran. Normalmente no es necesaria utilizar la opción La opción , pero puede ser útil en casos donde se realizó una limpieza manual de los valores del punto (por ejemplo, utilizando una funcionalidad de SQL).
