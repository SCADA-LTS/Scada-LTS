Los atributos utilizados para configurar los puntos de origen de los datos SQL difieren ligeramente dependiendo de si la consulta se basa en filas o columnas. En ambos casos, el tipo de datos Tipo de datos determina el tipo de datos del punto de datos Scada-LTS. Los valores del conjunto de resultados se convierten automáticamente a este tipo.

En las consultas basadas en filas, el identificador de fila Identificador de fila se compara con la primera columna del conjunto de resultados para encontrar la(s) fila(s) que contiene(n) valores (la segunda columna) para ese punto. Los rebasamientos de tiempo se toman de la tercera columna si existe.

En las consultas basadas en columnas, el Nombre de la columna es el nombre de la columna del conjunto de resultados de la que se obtiene el valor del punto. La columna Undo Time (opcional) especifica la columna que se usará como punto de deshacer el tiempo. Tenga en cuenta que varios puntos pueden utilizar el mismo tiempo de anulación si es necesario.

Actualización de las declaraciones

Una Declaración de actualización puede ser definida posiblemente para un punto, lo que determina cómo el valor del punto puede ser establecido en la base de datos. Esta es una declaración SQL estándar que utiliza "actualizar" o "insertar". Si no se define esta declaración, el punto se considerará como no fijable.
