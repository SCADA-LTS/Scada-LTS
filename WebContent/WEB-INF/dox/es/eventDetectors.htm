Visión general

El propósito de un detector de sucesos es determinar si el valor de un punto satisface una o más condiciones conexas y, en caso afirmativo, "activarse" y desencadenar un suceso que pueda manejarse adecuadamente. Un punto puede tener un número ilimitado de detectores de eventos. La forma en que se pueden detectar los eventos depende del tipo de datos del punto, por lo que sólo los detectores de eventos que se aplican al tipo de datos del punto actual se enumeran en la selección Tipo.

Los detectores de eventos pueden detectar un gran número de condiciones, pero sólo se aplican a un único punto. Si se requieren condiciones multipunto o condiciones con eventos programados, pueden utilizarse detectores de eventos compuestos. Vea la documentación sobre los detectores de eventos compuestos.

Un punto puede tener tantos detectores como sea necesario, y puede tener varios detectores del mismo tipo. (es decir, varios detectores de límite alto o bajo).

Para añadir un nuevo detector, seleccione el Tipo de la lista y haga clic en el icono . Tenga en cuenta que el botón "Guardar" debe ser pulsado para que el detector se guarde realmente con el punto. Para borrar un detector existente, haga clic en el icono asociado con el detector.

Cada tipo de detector tiene un atributo Nivel de alarma. Este atributo determina si el detector dispara una alarma cuando está activo, y si es así, la gravedad de la alarma. Consulte la página de ayuda general para obtener información sobre las alarmas.

Límite superior

Este detector se aplica sólo a los puntos numéricos. El Límite Alto es el valor por encima del cual el detector se activa. El detector puede configurarse opcionalmente con un Duración; el valor del punto debe permanecer por encima del límite superior durante al menos el tiempo dado antes de que el detector se active. Los eventos desencadenados por este detector volverán automáticamente a la normalidad cuando la condición ya no se cumpla.

Límite inferior

Este detector se aplica sólo a los puntos numéricos. El Límite inferior es el valor por debajo del cual el detector se activa. El detector puede configurarse opcionalmente con un Duración; el valor del punto debe permanecer por debajo del límite inferior durante al menos el tiempo dado antes de que el detector se active. Los eventos desencadenados por este detector volverán automáticamente a la normalidad cuando la condición ya no se cumpla.

Cambiar

Este detector es aplicable a todo tipo de datos. Desencadena un evento cada vez que cambia el valor del punto. Los eventos activados por este detector no vuelven automáticamente a la normalidad.

No hay cambios.

Este detector es aplicable a todo tipo de datos. Desencadena un evento cuando el valor no ha cambiado dentro de los límites de los datos Duración. Esto puede ser útil para detectar un mal funcionamiento del equipo. Los eventos activados por este detector volverán automáticamente a la normalidad cuando la condición ya no se cumpla.

No hay actualizaciones

Este detector es aplicable a todo tipo de datos. Se desencadena un evento cuando el valor no ha sido actualizado en el período de datos Duración. (Un cambio en el valor del punto requiere que el nuevo valor del punto sea diferente del anterior. Una actualización del valor puntual sólo requiere el tiempo para que el valor puntual cambie; el valor en sí mismo puede permanecer igual). Esto puede ser útil para detectar un mal funcionamiento del equipo. Los eventos activados por este detector volverán automáticamente a la normalidad cuando la condición ya no se cumpla.

Estado

Este detector se aplica a puntos binarios, multiestatales y alfanuméricos. Se activa tan pronto como el punto entra en el estado dado . El detector puede configurarse opcionalmente con un Duración; el punto debe permanecer en el estado dado durante al menos la duración dada antes de que el detector se active. Los eventos desencadenados por este detector volverán automáticamente a la normalidad cuando la condición ya no se cumpla.

CUSUM positivo

Este detector se aplica sólo a los puntos numéricos. El Límite Positivo es el CUSUM por encima del cual el detector se activa, y el Pesos es el valor de peso constante a utilizar para el cálculo del CUSUM. El detector puede configurarse opcionalmente con un Duración; el CUSUM debe permanecer por encima del límite durante al menos el tiempo dado antes de que el detector se active. Los eventos desencadenados por este detector volverán automáticamente a la normalidad cuando la condición ya no se cumpla.
