Descripción General

Los Scripts son procedimientos que automáticamente ejecutan tareas, habilitando el control de las fuentes de datos y de los puntos. Los Scripts no retornan un valor al finalizar, pero permiten establecer el valor de un punto (si es modificable).

JavaScript / ECMAScript

Los Scripts requieren de algún conocimiento de JavaScript, al igual que de ECMAScript.(Las especificaciones completas de ECMAScript se encuentran en aquí.) Para muchos, JavaScript es el lenguaje más popular para la programación de scripts de hoy en día, es el único lenguaje disponible para todos los navegadores populares. Como resultado esta popularidad es el uso común del JavaScript en las páginas webs. El contexto dentro de las fuente de datos Meta es diferente, no hay objetos "ventana" o "documentos", además de otras diferencias. Junto a las diferencias contextuales, una implementación completa de ECMAScript está disponible en las fuentes de datos. Específicamente la implementación Rhino es utilizada.

Configuración del Script

El contexto del Script define los puntos que estarán disponibles para el script en el momento que el se ejecute. Cada punto que es añadido debe tener un nombre de variable Var, que será el nombre con el que se le identificará dentro del script. Ese nombre var debe ser un nombre válido para variables ECMAScripts: Debe comenzar con cualquier letra o con un underscore (_) ,y no podrá contener espacios (pueden haber otras restricciones). Cuando defina un nombre de variable para ser usado dentro del script, recibirá una validación o una excepción de ejecución del script si no está correctamente definido. Para adicionar un punto en el contesto, selecciónelo de la lista y haga clic en el icono . Para eliminar un punto existente del contexto haga clic en el icono que está junto al punto a eliminar. Los puntos que no son necesarios en el script no deberían ser agregados al contexto ya que puede implicar sobrecarga en la preparación de los datos de preparación del script . De igual forma, variables innecesarias en el script pueden ocasionar ejecuciones no esperadas del script. (Vea "Ejecutando un script") De cualquier forma, lo contrario también puede ser cierto: puede ser útil incluir en el contexto del script una variable para que ese se ejecute.

Los objetos de contexto son para el control de una fuente de datos o de sus puntos. Entonces en el contexto de los puntos, un nombre de variable Var debe configurarse, y su correspondiente habilitación debe ser activada.

Ejecutando un Script

Su ejecución puede ser manual, al hacer clic en el icono , o automática, utilizando manejadores de eventos. Al realizar una ejecución manual se genera un mensaje indicando que se realizó correctamente o que ocurrió un error. Para poder ejecutar un script debe haberlo guardado primero.
