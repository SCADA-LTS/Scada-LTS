Vue d'ensemble

Les liaisons de point sont utilisées pour mettre à jour des points cibles à partir de valeurs de points sources. Les points seront synchronisés selon des règles définies par l'utilisateur. En pratique une liaison de point peut-être utilisée pour lire les valeurs d'un système (d'une ou plusieurs source de données) et les affecter immediatement à un autre système (à une ou plusieurs source de données).

On peut obtenir le même résultat en créant un détecteur qui observer les modifications ou mise à jour d'un point source, puis en créant un gestionnaire d'évènement de la "consigne" pour chaque point de destination définis. Cependant il est péférable d'utiliser les liaisons de point pour pluseurs raisons:

  • Une liaison de point ne crée pas d'évènement,ce qui préserve les ressources systèmes et améliore donc les performances
  • Les liaisons de point permettent définir des scripts, laissant à l'utilisateur une plus grande liberté
  • L'interface de cette fonction est contenue sur une seule page, ce qui la rend plus facile à utiliser

Note: Une liaison de point peut être confiqurée de telle sorte que des boucles infinies peuvent apparaître. Scada-LTS détectera le problème,le terminera et informera l'utilisateur.

Liste des liaisons de point

Pour ajouter une liaison de point, cliquez sur l'icône .Pour modifier une liaison, cliquez sur le lien dans la liste.

Configurer une liaison de point

L'Export ID est la valeur utilisée pour identifier une liaison dans les différentes instances de Scada-LTS. Il est utilisé dans le menu Import/Export. Cette valeur, unique est générée aléatoirement, mais elle peut être définie par l'utilisateur pour être plus explicite.

Le point source est le point dont la valeur sera suivie. C'est le point qui définira les valeurs cibles.

Le point cible est le point dont la valeur sera affectée.

Le Script est facultatif il transformera les valeurs sources si nécessaire. Des informations supplémentaires sur les scripts sont disponibles dans l'aide des propriétés des points Meta. Le nom de la variable dans le contexte est "source". un script tel que, "return source.value / 2;" attribuera a la valeur cible, la moitié de la valeur source. Notez que l'on peut aussi transformer le type de donnée ,exemple "if (source.value > 0) return true; return false;", ce qui convertira une valeur source numérique en une valeur cible binaire, Cliquez sur l'icône pour valider votre script (le point source doit être disponible).

Evènement va définir le type de mise à jour du point cible, à chaque mise à jour du point source (i.e. un nouveau timestamp n'engendre pas forcément une nouvelle valeur), ou seulement au changement de valeur(i.e. nouveau timestamp et nouvelle valeur).

Une liaison de point peut être Désactivée. Utile si vous ne voulez pas la supprimer et ne pas l'utiliser temporairement.

Pour enregistrer une liaison de point, cliquez sur l'icône en haut à droite. Pour supprimer une liaison de point, cliquez sur l'icône .
