Обзор

Пользователи – это люди, которые работают в системе. Пользователям могут назначаться различные права, чтобы предоставить им возможность чтения и изменения данных и настройки системы.

Существует два основных уровня пользователей: администраторы () и другие пользователи (). Администраторы имеют доступ ко всему функционалу системы. Другие пользователи обладают различными правами в зависимости от конфигурации системы.

Редактирование пользователей

Чтобы добавить нового пользователя, нажмите на пиктограмму . Пользователь добавляется в список только после его сохранения. Чтобы редактировать существующего пользователя, нажмите на имя пользователя в списке пользователей.

Пароль после сохранения не выдается; его можно только переустановить, заменив новым паролем.

Отключенный пользователь не имеет права входа в систему. Чтобы удалить пользователя, нажмите пиктограмму .

Чтобы проверить электронный адрес пользователя, нажмите пиктограмму . В этом случае на указанный электронный адрес будет направлено тестовое электронной сообщение.

На электронные адреса пользователей могут рассылаться предупреждения о тревожной ситуации, даже если эти адреса не были явно указаны в обработчике событий электронной почты. Если установить уровень тревоги функции Отправлять сообщения о тревожной ситуации выше значения «Нет», то пользователю будут рассылаться уведомления о возникновения события с уровнем тревоги выше или равным минимальному уровню тревоги пользователя. Уведомления рассылаются только при возникновении события (но не при эскалации или прекращении события). Обратите внимание, что, если пользователь уже указан в обработчике событий электронной почты, ему не будут отправляться групповые уведомления.

Полномочия пользователей

Флажок Администратор назначает пользователю права администратора.

Если пользователь не является администратором, в списке будут выводиться все источники данных, доступные для обычного пользователя. Если отметить флажком окошко рядом с именем источника данных, это будет означать, что пользователь может изменять настройки источника данных, а также будет иметь доступ к данным, поставляемым этим источником.

Если источник данных не отмечен флажком, будет отображен список всех точек источника. Для каждой точки можно индивидуально настроить доступ опциями: нет доступа, только чтение, или запись (если точка устанавливаемая). (Подразумевается, что права на запись предполагают права на чтение)
