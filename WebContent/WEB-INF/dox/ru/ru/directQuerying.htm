ScadaBR дает возможность прямого доступа к базе данных системы как для осуществления запросов, так и для обновления значений. Эта функция используется в первую очередь для устранения неполадок и восстановления системы. Она не подходит для регулярного использования из-за высокого риска потери или порчи данных.

Для выдачи запросов используются только правильные индивидуальные выражения на языке SQL. Нельзя заканчивать выражения символом «;».

Доступные таблицы

Ниже приводится список всех таблиц в базе данных ScadaBR:

  • compoundEventDetectors (Составные детекторы событий)
  • dataPoints (Объекты данных)
  • dataPointUsers (Пользователи объектов данных)
  • dataSources (Источники данных)
  • dataSourceUsers (Пользователи источников данных)
  • eventHandlers (Обработчики событий)
  • events (События)
  • mailingListInactive (Список рассылки сообщений об прекращении события)
  • mailingListMembers (Участники списка рассылки)
  • mailingLists (Списки рассылки)
  • maintenanceEvents (События обслуживания)
  • mangoViews (Виды Mango)
  • mangoViewUsers (Пользователи видов Mango)
  • pointEventDetectors (Детекторы событий объектов)
  • pointHierarchy (Иерархия объектов)
  • pointLinks (Связи объектов)
  • pointValueAnnotations (Комментарии значений объектов)
  • pointValues (Значения объектов)
  • publishers (Издатели)
  • reportInstanceData (Данные экземпляра отчета)
  • reportInstanceDataAnnotations (Комментарии данных экземпляра отчета)
  • reportInstanceEvents (События экземпляра отчета)
  • reportInstancePoints (Объекты экземпляра отчета)
  • reportInstanceUserComments (Комментарии пользователя экзмепляра отчета)
  • reportInstances (Экземпляры отчета)
  • reports (Отчеты)
  • silencedEvents (Заглушенные события)
  • scheduledEvents (Запланированные события)
  • systemSettings (Системные установки)
  • users (Пользователи)
  • userComments (Комментарии пользователей)
  • watchListPoints (Объекты листа наблюдения)
  • watchLists (Листы наблюдения)
