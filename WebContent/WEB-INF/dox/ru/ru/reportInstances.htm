Обзор

Отчеты в ScadaBR имеют два представления: шаблоны, которые определяют содержание отчета и экземпляры, которые являются результатом работы отчета по заданному шаблону. Проще говоря, механизм создания отчетов формирует экземпляры отчета из шаблонов отчета. Шаблон имеет следующие атрибуты: имя, список включенных точек и диапазон значений, которые он извлекает. Экземпляр отчета наследует свое имя из шаблона, но имеет собственные атрибуты, такие как время его работы, продолжительность работы, количество записей, и собственно данные.

Экземпляр можно считать моментальным снимком данных в определенный момент времени. Атрибуты экземпляра изменить нельзя. Кроме того, данные, которые хранятся в экземпляре отчета, существуют столько, сколько существует экземпляр. (Например, если он сохранился при очистке данных). По этой причине, очень важно производить очистку экземпляров отчета, особенно больших, если они больше не нужны, так как они могут занимать достаточно много места в хранилище.

Шаблоны отчета, и экземпляры, которые по ним создаются, принадлежат создавшему их пользователю. Они не могут находиться в системе в явном виде, но могут быть в совместном использовании с другими пользователями. Более подробная информация приводится в разделе «Шаблоны отчетов».

Очередь отчетов

В очереди отчетов перечислены все экземпляры отчетов текущего пользователя. Данный список не обновляется автоматически. Необходимо включить ручное обновление нажатием пиктограммы . Хотя при некоторых событиях, например, при запуске отчета, также выполняется обновление списка.
Экземпляры отчета приводятся в списке независимо от их состояния, были ли они завершены или все еще выполняются. Столбцы списка отражают текущее состояние экземпляра.

Столбцы списка имеют следующие заголовки:

  • Имя отчета: имя, наследуемое от шаблона отчета. После создания экземпляра, его имя не меняется, даже если изменилось имя шаблона.
  • Запуск: время начала создания экземпляра отчета.
  • Продолжительность работы: количество времени, затраченного на создание экземпляра.
  • С: время, начиная с которого были выбраны записи отчета (включительно).
  • До: время, до которого были выбраны записи отчета (исключающее)
  • Записи: общее количество выбранных записей.

The Do not purge checkbox allows users to prevent important report instances from being deleted by the scheduled purge process. (See the "Other settings" documentation for more information on report purging.) This feature should only be used when necessary since report instances can potentially consume large amounts of storage.

Флажок Не стирать позволяет пользователям запретить удаление важных экземпляров отчета процессом регулярной очистки. (Более подробную информацию об очистке отчетов смотрите раздел «Другие настройки»). Даная функция должна быть использована только при необходимости, так как экземпляры отчета могут занимать большой объем хранилища.

В последнем столбце таблицы приводятся управляющие элементы для работы с экземплярами. Пиктограмма запускает формирование CSV-файла отчета, который можно открывать в другом ПО для обработки электронных таблицах. Пиктограма открывает новое окно браузера с информацией об экземпляре отчета, статистикой и графиком, построенным на основании отчета. Нажатие на пиктограмму удаляет экземпляр отчета. Обратите внимание, что на удаление больших экземпляров может потребоваться значительное время.
