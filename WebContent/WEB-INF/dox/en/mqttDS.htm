MQTT (Message Queuing Telemetry Transport) - providing a lightweight publish/subscribe reliable messaging transport protocol suitable for communication in M2M/IoT contexts where a small code footprint is required and/or network bandwidth is at a premium.

Some technical information:

  • Protocol version: 3.1.1, 5.0

Data Source implementation based on: Eclipse Paho 1.2.5;

Configuration

Each MQTT Receiver Data Source require some parameters:

  • Name - Data Source unique display name
  • Server Host - Host of the broker to which we want to connect
  • Server Port - Port on which the broker listen to (default = 1883)

There are also some optional parameters:

  • Username - username required to login into the broker
  • Password - username password required to authenticate the user

ScadaLTS can also send a messages to related queue. It is as simple as it was for other DataPoint. Just open "watch list" menu select wrench button, type a new message which you want to send, and press link [send]. You can check your message on the other device.
