Configuration

Each datapoint configured into the Socket Communication datasource corresponding to a value received from the service, running at the Host defined into the datasource and listening to socket connections at the configured Port.

Every new point added to this datasource is identified by a Name and an internal identifier automatically generated, Export ID. There are two communication possibilities depending on how the server service provides the data:

  1. Command sending and response receiving
  2. Periodically data receiving

Command sending and response receiving

If the server needs to be queried to provide a data value, the first thing to do is establish the Number of commands needed to get the desired response from the server. If only one command is needed to get the sought response from the server, a First Command is sent. If two or three commands are needed to get the sought response from the server, the response to the first command is compared to the Expected response for first command. If both match, a Second command is sent and the same process is done with the Expected response for second command, if a Third Command is needed. If the server responses match the expected ones configured into the data point, a value from the final response (the sought one) is stored as as a binary, numeric, multistate or alphanumeric data (Data Type), based on the regular expression "mask" at Value regex.

Periodically data receiving

If the server sends data periodically without receiving any command from the datasource, the Number of commands field from each datapoint should be kept with a zero value, indicating that the datasource is configured to only receive data from the server service (No command is sent to the server).The final value to be stored as datapoint is selected based on a regular expression (Value regex). Depending on the Data type, the value obtained from the server is stored as a binary, numeric, multistate or alphanumeric data.
